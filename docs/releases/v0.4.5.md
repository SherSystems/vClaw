## Telegram approval-pending deep links.

Every approval-pending alert RHODES sends through Supra → Telegram now carries a deep link back to the dashboard with the plan id pre-targeted:

```
RHODES — approval needed
Action: qm delsnapshot 200 autosnap_2026-04-18 [risky_write]
…
Approve at: http://100.73.129.96:7412/?plan=plan_01HXYZ…
```

Tap the link on your phone, the dashboard opens, scrolls straight to the matching Pending Approvals card, and pulses a Rhodes Blue ring around it for two seconds. No more "find the dashboard URL from memory, scroll to find the right plan."

If the plan was already approved or rejected by the time you tap (a coworker beat you to it; you came back from dinner), the dashboard shows a one-line toast — `Plan plan_01HXYZ… has been approved.` — instead of pointlessly scrolling to a card that's gone. The deep-link handler runs on initial page load **and** after every Pending Approvals re-render, so if you tap the link before the SSE has delivered the awaiting_approval entry, the highlight still fires the moment the card appears.

**What this means for you:** approve from your phone in three seconds, not the thirty it takes to remember the IP, open the page, and hunt for the right card.

## Configuration

Set `RHODES_DASHBOARD_URL` in `.env`. Empty (the default) suppresses the link entirely — alerts fall back to the previous "Plan: <id>" line so nothing breaks on misconfigured installs.

```bash
# Example for the NUC over Tailscale
RHODES_DASHBOARD_URL=http://100.73.129.96:7412
```

The deep link is built as `<RHODES_DASHBOARD_URL>/?plan=<plan_id>` with any trailing slash on the base URL normalised away.

## What's _not_ in this release

Inline `Approve` / `Reject` buttons in the Telegram message itself — that requires a signed-callback flow through Supra's bot webhook (~3-5 hours of work). Deep link first; inline buttons next.
